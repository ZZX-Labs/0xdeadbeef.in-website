Publication Archive

Writing, research, and field documentation.

This blog serves as the public journal for ongoing work by 0xdeadbeef. It combines development logs, technical articles, research notes, whitepapers, project updates, release announcements, travel journals, field observations, and long-form commentary.

Posts are stored as plaintext files whenever possible so the archive remains portable, inspectable, versionable, and easy to preserve.

Each post directory contains its own page, metadata record, and plaintext source. The post page loads the local text file and renders it into the shared publication layout.

Cryptographic signatures, hashes, downloadable formats, source records, and revision history may be added to individual posts when those verification materials exist.

Plaintext-first publishing

The canonical article body is stored in post.txt. Post metadata is stored in metadata.json, while the public archive index is maintained in /blog/blog.json.

Publication System

How a post is published.

Stage 01

Create the post directory

Create the next sequential directory under /blog/blog_posts/, such as post_0002/.

Stage 02

Create metadata.json

Define the post number, title, subtitle, summary, publication date, update date, author, category, tags, status, URLs, and publication state.

Stage 03

Write post.txt

Place the canonical article body in the plaintext file. Separate paragraphs with blank lines and preserve the original textual record.

Stage 04

Create the post page

Use the shared post layout to load metadata.json and post.txt from the same directory.

Stage 05

Add the record to blog.json

Add one post object to the master publication index. The blog page will generate the card, filters, links, and search data automatically.

Stage 06

Validate over HTTP or HTTPS

Test the page through the web server because browser security policies generally prevent local fetch() requests from working through file://.
