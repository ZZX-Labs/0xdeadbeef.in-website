Publication Archive

Writing, research, and field documentation.

This blog serves as the public journal for ongoing work by 0xdeadbeef. It combines development logs, technical articles, research notes, whitepapers, project updates, release announcements, travel journals, field observations, and long-form commentary.

Posts are stored as plain-text files whenever possible so the archive remains portable, inspectable, versionable, and easy to preserve.

Each post directory contains its own page, metadata record, and plaintext source. The post page loads the text file locally and renders it into the shared publication layout.

Cryptographic signatures, hashes, downloadable formats, and revision records may be added to individual posts when those verification materials exist.

Plaintext-first publishing

The canonical article body is stored in post.txt. Metadata is stored separately in metadata.json. The page remains usable without a server-side application.

Publication System

How a post is published.

Stage 01

Create the directory

Copy post_template/ to a new directory named post_0002/, post_0003/, or the next sequential post number.

Stage 02

Edit metadata.json

Set the post number, title, subtitle, date, category, tags, summary, status, author, and optional verification details.

Stage 03

Write post.txt

Place the complete article body in the plaintext file. Separate paragraphs with blank lines. Lightweight headings and lists are supported.

Stage 04

Add the post to blog.json

Add one metadata object to the blog index. The main blog page will generate the post card automatically.

Stage 05

Test over HTTP

Use a local web server because browsers normally block fetch() when HTML is opened directly through file://.
