Publication Format

Plaintext-first and independently readable.

The canonical article body is stored in post.txt. This page loads that file and renders supported plaintext formatting into the shared website layout.

The article remains directly accessible even if the rendering script fails because the source text can be opened independently.

Metadata is stored separately in metadata.json, allowing the title, publication date, category, tags, revision, license, hashes, signatures, and source links to be updated without changing the page structure.

The same post layout is reused across the complete blog archive through /static/js/blog-post.js.
